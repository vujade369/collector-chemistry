# Constellate Interface Craft

## Purpose

Use this skill for UI hierarchy, page modules, component presentation, copy, labels, empty states, user-facing claims, and display/link behavior.

## Product feel

Constellate should feel:

- interpretive and behavior-led
- human and editorial
- restrained, legible, and useful
- proof-aware without becoming forensic

Avoid:

- generic dashboards
- marketplace framing as the center of the experience
- ranking, status, compatibility, alpha, or portfolio-score language
- unsupported personality or taste claims
- noisy metrics without a clear read

## Interface rules

- Name the primary read before changing a surface.
- Put visible proof near interpretive claims.
- Separate facts from interpretation.
- Use careful language when data is incomplete.
- Preserve truthful loading, empty, and error states.
- Preserve accessibility basics for links, controls, images, and state changes.
- Every displayed NFT, collection, or artist should link to the deepest reliable destination available.
- Do not invent OpenSea URLs; prefer no link over a wrong link.

## Boundaries

- Do not touch API routes, data fetching, offer math, resolver behavior, or response contracts during an interface-only pass.
- Use existing components, CSS patterns, and page structure unless the scope explicitly permits broader hierarchy work.
- If interface guidance conflicts with a legacy skill or product doc, stop and report the conflict.
